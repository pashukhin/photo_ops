# web

## Local context

- Next.js app for authenticated upload + the rich photo gallery; runs on `WEB_PORT` (default 3000).
- Calls api-gateway via `NEXT_PUBLIC_API_BASE_URL` (default `http://localhost:3001`); all HTTP calls are in `lib/api.ts` (`listPhotos(params) → {photos, totalCount}`, `getPhoto(id)`, auth + upload).
- App shell & routing (session 014): the app has a shared shell + auth boundary. `lib/session.tsx` (`SessionProvider` + `useSession()`) is the single source of the current user — it fetches `getCurrentUser` on mount into `{ user, status }` (`'loading'|'anonymous'|'authenticated'`) and owns `login`/`signUp`/`logout`/`refresh`; a stale mount fetch cannot clobber a later mutation (guarded by a request-id ref). Routes: `/` → `redirect('/photos')`; the `(app)` route group (`app/(app)/layout.tsx`) wraps its pages in `AuthGuard` (anonymous → `router.replace('/login')`; renders children only when authenticated) then `AppShell` (`components/shell/`: top bar — brand → `/photos`, nav Photos·Clusters·Usage with `aria-current="page"` on the active link via `usePathname`, user menu + Log out — a failed logout surfaces a `role="alert"` banner); the `AuthGuard` `loading` branch renders a non-blocking `role="status"` affordance (not a blank screen), and `SessionProvider` `console.warn`s a failed `getCurrentUser` (session 015, `photo_ops-56l`); `/photos` = `components/photos/PhotosPage` (gallery + the upload action moved from the old home page), `/clusters`, `/usage` sit in the group. `/login` is OUTSIDE the group (bare `LoginScreen` in `components/auth/`; redirects an already-authenticated visitor → `/photos`). Root `layout.tsx` wraps everything in `SessionProvider`. The session-001 `.panel` CSS is retired.
- Gallery (session 011) lives in `components/gallery/`: `PhotoGallery` (container — owns the query state, fetches via `listPhotos`, polls every `GALLERY_POLL_MS` while a photo is uploading/processing, refetches on `reloadToken` change) composes `GalleryToolbar` (search/status/sort), `PhotoTable`, `GalleryPagination`, and `PhotoDetailModal` (lazy-mounted; re-fetches `getPhoto` for a fresh presigned preview, shows the error in-dialog on failure). Sort/filter/pagination are server-side. `app/page.tsx` renders `<PhotoGallery reloadToken>` and bumps the token after an upload. Poll hardening (session 015, `photo_ops-gfs`): a transient poll error no longer stops polling (tolerates `GALLERY_POLL_MAX_ERRORS` consecutive failures), a never-settling status stops after `GALLERY_POLL_MAX_TICKS`, and a stale in-flight poll response is dropped via a generation ref bumped on every main fetch (all in `types.ts`). Shared display formatters (`FALLBACK`/`fmt`/`fmtBytes`/`fmtDimensions`) live in `components/gallery/format.ts`, imported by `PhotoTable` + `PhotoDetailModal`.
- UI library: Tailwind (v4, CSS-first in `app/globals.css`) + shadcn/ui primitives vendored under `components/ui/` (`cn` in `lib/utils.ts`). Theme (session 015, `photo_ops-zv7`): a single coherent light theme — the shadcn tokens in `:root` are gently warmed (soft off-white `--background`, softer ink, `--radius` 0.625rem); the session-001 dark `body` override was removed (it fought the light tokens). The `.dark` token block is defined but inert (no `html.dark`); a dark-mode toggle is out of scope. `next build` does not run ESLint (`next.config.js` `eslint.ignoreDuringBuilds`); the one canonical lint is root `make lint`.
- Upload flow: browser calls `POST /photos/upload-intents` on api-gateway to get a presigned `uploadUrl`, then does a direct `PUT` to that MinIO URL (`uploadFileToPresignedUrl`), then calls `POST /photos/{id}/complete-upload` on api-gateway to finalize.
- Clustering UI (session 013) lives in `components/clusters/`: `ClusterView` (container — fetches `listClusteringResults` + `listClusteringMethods`, offers a method picker + Generate which `generateClusters` then polls `getClusteringResult` every `CLUSTER_POLL_MS` until it leaves `pending` (bounded by `CLUSTER_POLL_MAX_ATTEMPTS` — a stuck-PENDING run surfaces a timeout error instead of spinning forever, session 015 `photo_ops-n7w`), and renders a chosen result's immutable tree as a nested list). `app/clusters/page.tsx` renders `<ClusterView />`. Clustering client calls live in `lib/api.ts`.
- Tests: `vitest run` (`make test-web`) in jsdom with `@testing-library/react` (config `vitest.config.ts`, setup `vitest.setup.ts`); component behavior (render, click, modal, polling with fake timers) is exercised there. The live UI smoke (`smoke/gallery.smoke.ts`, Playwright) runs via `make smoke-ui` against a running stack — excluded from vitest, not part of `make gate`.
- Typecheck: `tsc --noEmit` (`make typecheck` runs it across all services).

## Local invariants

- Talks only to `api-gateway`, except for presigned MinIO upload URLs (direct browser-to-MinIO PUT).
- Does not hold its own database or business state; the gateway is the source of truth.
- Auth reaches the app only through `useSession()`; `SessionProvider` (`lib/session.tsx`) is the ONLY caller of the `lib/api` auth endpoints (`getCurrentUser`/`login`/`signUp`/`logout`). Components never import those directly.
