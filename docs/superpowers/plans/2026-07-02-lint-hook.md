# Edit-time lint hook (8d5) Skeleton Commit

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to fill this skeleton task-by-task ‚Äî each task makes its RED tests green within the provided stubs. Steps use checkbox (`- [ ]`) syntax.

**Goal:** A `PostToolUse` (Write|Edit) hook `scripts/lint-changed` runs a fast per-file linter on the edited file and feeds any issues back to the agent (exit 2); clean/unknown/opted-out ‚Üí exit 0.

**Architecture / WHY:** A pure dispatch `linter_for(path) -> Linter(cmd, issue_on) | None` (unit-tested) maps a file suffix to its fast linter + how to detect issues; a thin python3 orchestrator `scripts/lint-changed` reads the hook stdin JSON, honours `LINT_HOOK=0`, runs the command, and maps issues ‚Üí exit 2. Entry points: core ‚Üí `scripts/linthook/dispatch.py`; orchestrator ‚Üí `scripts/lint-changed`; behaviour ‚Üí `scripts/linthook/tests/`; wiring ‚Üí `.claude/settings.json`. Design note: `docs/superpowers/specs/2026-07-02-lint-hook-design.md`.

**Tech Stack:** Python 3 stdlib (`json`, `subprocess`, `sys`); eslint / ruff / gofmt as the per-file linters. Unit tests via the existing `scripts/coverage/.venv` (pytest).

## Global Constraints

- Linters by suffix: `*.ts`/`*.tsx` ‚Üí `pnpm exec eslint <file>` (issue_on=exit); `*.py` whose path contains `apps/media-worker/` ‚Üí `apps/media-worker/.venv/bin/ruff check <file>` (issue_on=exit); `*.go` ‚Üí `gofmt -l <file>` (issue_on=output). Any other path (incl. non-media-worker `.py`) ‚Üí skip.
- Issue detection: `issue_on="exit"` ‚Üí non-zero return code is an issue; `issue_on="output"` ‚Üí non-empty stdout is an issue.
- On issue: print the linter output to stderr, exit 2. Clean / skip / `LINT_HOOK=0` / missing-or-unparseable stdin / no file_path / linter tool absent ‚Üí exit 0.
- Excluded: `tsc`, `mypy`, `golangci-lint`, `make gate`, and auto-fix. Never run whole-project.
- The orchestrator must resolve `from scripts.linthook.dispatch import ...` regardless of CWD (insert repo root ‚Äî computed from `__file__` ‚Äî into `sys.path`).
- Core public names must not start with `test_` (pytest collection): `linter_for`, `Linter`.

## Non-Goals

- Not tsc/mypy/golangci-lint/make gate; not auto-fix; not a correctness gate (advisory). Non-media-worker `.py` unlinted.
- The hook takes effect NEXT session (settings hooks load at session start) ‚Äî not runtime-tested here.

```python
import json
import subprocess
from pathlib import Path

SCRIPT = Path(__file__).resolve().parents[2] / "lint-changed"  # scripts/lint-changed
BAD_GO = "package x\nfunc  F(){\n}\n"   # gofmt-misformatted (double space)
GOOD_GO = "package x\n\nfunc F() {}\n"  # gofmt-clean


def _run(file_path, env=None):
    payload = json.dumps({"tool_input": {"file_path": str(file_path)}})
    return subprocess.run([str(SCRIPT)], input=payload, capture_output=True,
                          text=True, env=env)


def test_misformatted_go_reports_and_exits_2(tmp_path):
    # why: an issue on the edited file must feed back to the agent (exit 2, named).
    f = tmp_path / "bad.go"
    f.write_text(BAD_GO)
    r = _run(f)
    assert r.returncode == 2
    assert "bad.go" in (r.stderr + r.stdout)


def test_clean_go_exits_0_silently(tmp_path):
    # why: a clean file must not interrupt the agent.
    f = tmp_path / "good.go"
    f.write_text(GOOD_GO)
    assert _run(f).returncode == 0


def test_lint_hook_disabled_exits_0(tmp_path):
    # why: LINT_HOOK=0 is the session escape valve ‚Äî no linting.
    import os
    f = tmp_path / "bad.go"
    f.write_text(BAD_GO)
    env = {**os.environ, "LINT_HOOK": "0"}
    assert _run(f, env=env).returncode == 0


def test_unknown_suffix_exits_0(tmp_path):
    # why: non-source files are a no-op.
    f = tmp_path / "notes.md"
    f.write_text("# hi")
    assert _run(f).returncode == 0
```
