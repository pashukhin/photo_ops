# Clustering Backend Skeleton Commit

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to fill this skeleton task-by-task — each task makes its RED tests green within the provided stubs. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deterministic space-time photo clustering end to end — a user generates clusters over their `ready` photos and reads back a persisted, immutable tree (date ranges, cover, counts, `not_clusterable` bucket, spacelike-anomaly flags) via `api-gateway`.

**Architecture / WHY:** New `cluster-service` (Python) owns `cluster-db`; async over RabbitMQ (`result_id == job_id`); compute = Euclidean scaled space-time metric → agglomerative (scipy) dendrogram-as-tree → causal overlay → `not_clusterable` bucket → self-metered consumption. Entry points: why → `docs/adr/0004-deterministic-space-time-clustering.md` + `docs/adr/0005-immutable-clustering-result-snapshots.md`; contracts → `proto/cluster/v1/*`, `proto/photo/v1/photo_service.proto`; behavior → the RED test files below; invariants → each service's `## Local invariants`.

**Tech Stack:** Python 3.12 (scipy, numpy, pika, grpcio, psycopg) mirroring `media-worker` + `photo-service` patterns; TS/NestJS for `photo-service` + `api-gateway`; PostgreSQL; RabbitMQ; proto-first via `buf`/`make proto`.

## Global Constraints

- `cluster-service` owns and connects only to `cluster-db`; it never connects to `photo-db`. Photo space-time data comes from `photo-service` via gRPC (`ListPhotoSpacetime`).
- Cross-service references are UUID v7; no cross-service foreign keys.
- Only `cluster-service` writes `cluster-db`; the worker has no DB and reports results over the queue.
- Async transport mirrors the `media-worker` RabbitMQ topology exactly (durable exchanges/queues, DLX/DLQ, ack-on-success). Do not invent a second async mechanism.
- Determinism: stable input order (`taken_at`, then `photo_id`); identical (input set + parameters) ⇒ identical tree structure + membership. Node/item ids are per-run UUID v7.
- Python tooling mirrors `media-worker`: `pyproject.toml`, `.venv`, `ruff` + `mypy` + `pytest`; proto stubs generated by `make proto` (committed, excluded from ruff/mypy).
- No testcontainers in this session (`photo_ops-4vg`); raw SQL is verified by `make smoke-*` + manual e2e, not in-process DB tests. Pure/mapping logic is unit-tested.
- Defaults (overridable per request): `c_kmph=5`, `v_max_kmph=1000`, `linkage=average`, `algorithm=agglomerative`.

## Non-Goals

Selectable algorithms (HDBSCAN/OPTICS/BIRCH); `scope = query | explicit_set` (only `all`); soft-delete/restore *operations* (column only); carry-over + cluster-equivalence; causal structure as the primary criterion (overlay flag only); notes / `notes-service`; reverse geocoding / admin labels; tz resolution of mixed-tz timestamps; usage *pricing* / `usage-service` wiring; web UI. (See ADR 0004/0005 negative space.)

from datetime import datetime
from cluster_service.models import SpacetimePhoto
from cluster_service.clustering.metric import haversine_km, spacetime_distance
from cluster_service.clustering.bucket import split_eligibility

def test_haversine_known_distance():
    # why: great-circle km is the spatial term; ~157.2 km for 1° latitude apart at the equator
    d = haversine_km(0.0, 0.0, 0.0, 1.413)
    assert abs(d - 157.2) < 1.0

def test_spacetime_distance_combines_time_and_space():
    # why: d^2 = (c*dt_hours)^2 + d_geo^2; c=10 km/h, 2h apart, same place -> 20 km
    a = SpacetimePhoto("a", datetime(2026,1,1,0,0), None, 0.0, 0.0)
    b = SpacetimePhoto("b", datetime(2026,1,1,2,0), None, 0.0, 0.0)
    assert abs(spacetime_distance(a, b, c_kmph=10.0) - 20.0) < 1e-6

def test_split_eligibility_routes_missing_data():
    # why: photos without time or coords go to not_clusterable, run continues
    t = datetime(2026,1,1)
    ok = SpacetimePhoto("ok", t, None, 1.0, 2.0)
    no_gps = SpacetimePhoto("no_gps", t, None, None, None)
    no_time = SpacetimePhoto("no_time", None, None, 1.0, 2.0)
    clusterable, not_clusterable = split_eligibility([ok, no_gps, no_time])
    assert [p.photo_id for p in clusterable] == ["ok"]
    assert {p.photo_id for p in not_clusterable} == {"no_gps", "no_time"}
